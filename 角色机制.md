## 1. 通用机制

### 行动值（AV）计算

```
基础行动间隔 = 10000 / 当前速度
```

### 技能代码

| 代码 | 含义 |
|------|------|
| `""` | 普攻（默认） |
| `"A"` | 战技（A 类） |
| `"E"` | 战技（E 类） |
| `"Q"` | 终结技 |
| `"W"` | 特殊技能（仅白厄境界内可用） |
| `"F"` | 协战标记（附加在技能上，如 `"EF"`） |
| `"M"` | 迷迷拉条 |

技能可组合，例如 `"EW"`、`"EF"`。`F` 后缀表示该行动同时触发协战。`EE` 会被流萤燃烧状态拆分为两次独立 E。

### 角色数据格式

角色定义在 `src/data/characters.json`，包含：
- `cid`：角色 ID
- `names`：别名列表
- `path`：命途（Harmony / Nihility / Destruction / Hunt / Erudition / Preservation / Abundance / Remembrance）
- `semantics`：语义标记数组
- `effects`：技能效果映射
- `effectRules`：特殊机制规则

## 2. 流萤（Firefly）— 完全燃烧

### 触发条件
- 使用 **Q**（终结技）且角色拥有 `fireflyCombustion` 效果
- 不能在已处于燃烧状态时再次触发

### 效果
- **速度 +60**（持续至燃烧结束）
- 技能限制为：`""`（普攻）、`"A"`、`"E"`（仅限燃烧期间）
- 生成 **70 速度的倒计时**（完全燃烧倒计时），倒计时结束后燃烧结束
- **Q 后立即行动**：下一动被 100% 提前到 Q 的当前 AV（由 `activateCombustion` 设 `nextActionValue = actionValue`）

### 击破机制
- 燃烧期间的每个行动可标记为「击破」（通过 `fireflyBreakCounters[key] = true`）
- 每次击破触发：
  - **倒计时延后**：`10000 / 70 * 10% ≈ 14.29 AV`，最多延后 3 次
  - **E2 额外回合**（仅限 E2）：生成一个 `-break-extra-N` 行动在当前 AV
- 额外回合也可标记击破，进一步延后倒计时

### EE 拆分
- 如果技能为 `"EE"`（连续两次战技），会在燃烧激活时拆分为两个独立 E 行动

### 配置参数

| 参数 | 值 |
|------|-----|
| 倒计时名称 | `完全燃烧倒计时` |
| 倒计时速度 | 70 |
| 速度加成 | +60 |
| 可用技能 | `""`、`"A"`、`"E"` |
| 击破延迟比例 | 10% |
| 最大击破延迟触发 | 3 次 |
| E2 额外回合 | 每击破生成 1 个 |

### 相关文件
- `src/utils/fireflyCombustion.ts` — 核心逻辑
- `src/utils/simulateActions.ts` — 主循环中的击破检查

## 3. 阿格莱雅（Aglaea）— 衣匠 & 至高之姿

### 衣匠（Garmentmaker / 忆灵）

#### 召唤
- 使用 **E**（战技）召唤忆灵衣匠
- 衣匠基础速度 **35**

#### 行动
- 衣匠每次行动使用固定技能 `刺纹之陷`（不可更改）
- 每次行动后**层数 +1**
- 速度 = `35 + 55 × 层数`
- 最大层数：**6 层**（E4 时 **7 层**）

#### 遣散
- 倒计时结束时衣匠离场，保留 `min(1, 当前层数)` 层

### 至高之姿（Supreme Stance）

#### 触发
- 使用 **Q**（终结技）激活
- 如果已在至高之姿中，**刷新倒计时**

#### 效果
- 阿格莱雅速度 = `基础速度 × (1 + 0.15 × 层数)`
- 技能限制：`""`、`"A"`、`"Q"`
- 生成 **100 速度倒计时**（至高之姿倒计时）
- 倒计时结束 → 衣匠离场，姿态结束

### E4 机制
- 使用 A 或 Q 时额外增加一层衣匠层数
- 衣匠最大层数提升至 7 层

### 配置参数

| 参数 | 值 |
|------|-----|
| 衣匠名称 | `衣匠` |
| 衣匠技能 | `刺纹之陷` |
| 衣匠基础速度 | 35 |
| 每层速度加成 | 55 |
| 最大层数 | 6 |
| E4 最大层数 | 7 |
| 阿格莱雅速度比例/层 | 15% |
| 遣散保留层数 | 1 |
| 倒计时名称 | `至高之姿倒计时` |
| 倒计时速度 | 100 |
| 至高之姿可用技能 | `""`、`"A"`、`"Q"` |

### 相关文件
- `src/utils/aglaeaGarmentmaker.ts` — 核心逻辑

## 4. 白厄（Phainon）— 反击境界（CounterW）

### 触发
- 使用 **Q**（终结技）激活反击境界
- 角色需拥有 `counterW` 效果

### 境界机制
- 境界生成一系列**境界内行动**，按固定间隔排列
- 敌人可自然穿插行动（不强制对齐）
- 每一动可用技能：`""`、`"E"`、`"EW"`、`"EA"`、`"A"`、`"W"`（W 仅在境界内可用）
- 最后一动固定为 **Q**（终结技）

### 行动间隔计算

```
境界行动间隔 = 10000 / 基础速度 / 系数 / max(1, 额外行动数 - 1)
```

- 系数：普通 0.6，E1 0.66
- 基础速度可通过 `baseSpeed` 配置（默认 106.0）
- 默认额外行动数：**8**

### W/EW 触发敌人行动
- 境界内使用 W 或 EW 时，所有敌人目标获得一个额外行动（与当前 AV 相同）

### 境界结束
- 结束后，所有友方获得 `15% × 基础速度` 的速度加成（持续至下一次行动）
- 保持剩余行动距离不变，在新速度下重新计算

### 昔涟互动 — 无尽境界
- 昔涟对白厄施放「负世之诗」时，境界变为**无尽模式**
- 行动数不再固定为 8，而是延伸到模拟上限

### 配置参数

| 参数 | 值 |
|------|-----|
| 默认基础速度 | 106.0 |
| 普通系数 | 0.6 |
| E1 系数 | 0.66 |
| 额外行动数 | 8 |
| 结束速度加成 | 15% 基础速度 |
| 可用技能 | `""`、`"E"`、`"EW"`、`"EA"`、`"A"`、`"W"` |
| 终结技 | `"Q"` |
| 敌人触发技能 | `"W"`、`"EW"` |

### 相关文件
- `src/utils/phainonDomain.ts` — 核心逻辑

## 5. 昔涟（Cyrene）— 终极技·诗篇系统

### 触发
- 使用 **Q**（终结技）激活
- 角色需拥有 `cyreneUltimate` 效果

### 机制
- Q 后生成忆灵**德谬歌**的行动（技能固定为 `Q`）
- 德谬歌行动时，可选择向一个友方施放**诗篇**

### 诗篇列表

| 诗篇 | 代码 | 目标 | 持续 | 效果 |
|------|------|------|------|------|
| 诗篇（通用） | `generic` | 任意 | 2 回合 | — |
| 创世之诗 | `genesis` | 开拓者·记忆 / 记忆主 / RMC | 整场 | — |
| 门径之诗 | `passage` | 缇宝 / Tribbie | 整场 | — |
| 生死之诗 | `lifeDeath` | 遐蝶 / Castorice | 整场 | — |
| 诡计之诗 | `trickery` | 赛飞儿 / 猫 / 猫猫 / Cipher | 整场 | — |
| 负世之诗 | `worldbearing` | 白厄 / Phainon | 整场 | 白厄境界变为无尽模式 |
| 律法之诗 | `law` | 刻律德菈 / 刻律 / 刻 / Cerydra | 整场 | — |
| 岁月之诗 | `time` | 长夜月 / 月 / Evernight | 整场 | — |
| 大地之诗 | `earth` | 丹恒·腾荒 / 盾丹 / DHPT | 整场 | — |
| 理性之诗 | `reason` | 那刻夏 / 夏 / Anaxa | 下回合 | 立即获得一个回合 |
| 纷争之诗 | `strife` | 万敌 / Mydei | 额外回合 | 立即获得强化 E 回合 |
| 海洋之诗 | `ocean` | 海瑟音 / Hysilens | 下次攻击 | — |
| 天空之诗 | `sky` | 风堇 / Hyacine | 2 层 | 使用 E/Q 消耗层数 |

### 特殊诗篇效果
- `immediateTurn`：目标 `nextActionValue = currentActionValue`（立即回合）
- `immediateEnhancedSkill`：目标立即获得强化 E 额外行动（`isOdeExtraAction`）
- `endlessCounterWDomain`：白厄境界变为无尽模式

### 配置参数

| 参数 | 值 |
|------|-----|
| 忆灵名称 | `德谬歌` |
| 忆灵技能 | `Q` |
| 默认资源 | `追忆` |

### 相关文件
- `src/utils/simulateActions.ts` — 核心逻辑（`emitCyreneMemospriteAction`）

## 6. 记忆主（RMC）— 迷迷拉条

### 召唤
- 使用 **E**（战技）召唤忆灵**迷迷**
- 迷迷速度 **130**（固定）

### 拉条机制
- 任何角色行动后，如果在 `memeSelections` 中为该行动选择了目标，迷迷消耗自己的回合向目标施放拉条
- 拉条效果：**100% 行动提前**（目标 `nextActionValue = 当前 AV`）
- 拉条通过 `emitMemeAdvanceAction` 实现

### 配置参数

| 参数 | 值 |
|------|-----|
| 迷迷名称 | `迷迷` |
| 迷迷技能 | `拉条` |
| 迷迷速度 | 130 |
| 拉条比例 | 100% |

### 相关文件
- `src/utils/simulateActions.ts` — `emitMemeAdvanceAction`

## 7. 姬子·启行（Himeko Nova）— F 协战

### 触发
- 角色需拥有 `himekoNovaAssist` 效果
- 队友的技能中包含 `F` 后缀时触发（如 `"EF"`）

### 机制
- 协战行动与触发者在同一 AV 执行
- 每次触发最多 **2 次**协战
- 同一回合内只触发一次
- 协战行动标记为 `isAssistAction`，带 `assistSourceKey` 和 `assistIndex`

### 配置参数

| 参数 | 值 |
|------|-----|
| 触发条件 | 队友行动 |
| 排除自身 | 是 |
| 生成的技能代码 | `F` |
| 执行顺序 | 目标行动前、插队前 |

## 11. 知更鸟（Robin）— 全队顶轴

### 触发
- 使用 **Q**（终结技）
- 效果：`robinUltimate`

### 效果
- 知更鸟速度变为 **90**
- 下一动锁定为 `当前AV + 10000/90`
- `blockNextAdvance = true`（免疫拉条）
- 所有友方按速度顺序排列在 `当前AV + 微小偏移`（第一名当前 AV，第二名 +0.0001，依此类推）
- 首动提前 25%（`firstActionAdvance25`）

## 12. 忘归人（Fugue）— 全队 24% 拉条

### 触发条件
- 使用 **Q** 且角色有 E2（`hasEidolon2`）和 `teamAdvance24` 效果
- 或者有 `hasDance`（舞舞舞）且命途为 `Harmony`

### 效果
- 全队 24% 拉条（包括自身）
- 拉条量 = `2400 / 目标当前速度`
- 不提前于当前 AV：`newAV = max(当前AV, 原AV - 拉条量)`

### 战技（E）
- 效果：`allyTargetSelectable`（可选择目标）

## 15. 风套（Wind Set）— 大招后 25% 提前

### 触发条件
- 需标记 `hasWindSet: true`
- 使用 **Q**（终结技）

### 效果
- 大招后下一动提前 25%
- 提前量 = `2500 / 当前速度`

## 17. 语义标记

角色可以在 `characters.json` 中声明 `semantics` 数组，用于控制模拟器行为：

| 标记 | 含义 |
|------|------|
| `ambiguousBaseName` | 角色名称是其他角色名称的前缀，匹配时需精确匹配 |
| `noDefaultSkillTarget` | 不记录默认技能目标（昔涟） |
| `wOnlyInDomain` | W 技能仅在境界内可用（白厄） |
